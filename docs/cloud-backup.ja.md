# Markwell をクラウドにバックアップする

[English](cloud-backup.md) · [中文（台灣）](cloud-backup.zh-TW.md) · **日本語** · [한국어](cloud-backup.ko.md)

Markwell のライブラリを iCloud Drive・Google ドライブ・Dropbox・OneDrive に
置き、新しいコンピュータでもそのまま取り戻すためのガイドです。新しい
アカウントは不要で、必要なのは数クリックだけです。

## これでうまくいく理由

Markwell が保存するものはすべて、1 つの普通のフォルダに入った普通の
ファイルです。保存コピーは `backups/` に、読めるページとエクスポートは
`output/` に、作成した ZIP アーカイブもここにあります。クラウドサービスは
もともとフォルダの同期が得意です。だから Markwell はあなたのパスワードを
必要とせず、クラウドの API とも話さず、アカウント登録も求めません。
クラウドアプリがすでに同期しているフォルダの*中に* Markwell フォルダを
置くだけで、あとはクラウドアプリがやってくれます。

Markwell 自身は何ひとつアップロードしません。ネットワーク接続を一切行わず、
同期は、あなたがすでにほかのファイルを任せて信頼しているクラウドアプリ
だけが行います（[SECURITY.md](../SECURITY.md) 参照）。

## ライブラリをクラウドへ移す

まず、クラウドのデスクトップアプリ — Apple・Google・Dropbox・Microsoft が
提供する、コンピュータ上のフォルダを同期するアプリ — がインストール済みで
サインインしていることを確認してください。そのうえで、Markwell で：

1. サイドバーの**設定**を開きます。
2. 「**Markwell フォルダの置き場所**」に、このコンピュータで見つかった
   クラウドが選択肢として並びます。お使いのものを選びます。
3. 「**ここにライブラリを置く**」を押します。
4. これから起きることを Markwell が説明します — すべてのファイルは新しい
   フォルダに*コピー*され、元のファイルはそのまま残ります。
   「**コピーする**」を押します。

これで完了です。レポートには、コピーされた保存コピーとファイルの数、元の
ファイルが今も残っている場所が示されます。**何も削除されません。** 以後の
バックアップはすべてクラウドのフォルダに保存され、クラウドアプリが自動的に
アップロードします。

Markwell はライブラリを、クラウドフォルダの最上位にある `Markwell` という
名前のフォルダに置きます。どの端末からでも見つけやすい場所です。

## プロバイダごとのメモ

Markwell 内の手順はどのプロバイダでも同じです。違いは、どのデスクトップ
アプリが必要かだけです。

- **iCloud Drive** — macOS に組み込み。iCloud を使っていればすでに
  あります。Windows では Microsoft Store から **iCloud for Windows** を
  インストールし、iCloud Drive を有効にします。
- **Google ドライブ** — **パソコン版 Google ドライブ**をインストールして
  サインインすると、同期された Google ドライブの場所がコンピュータに
  現れます。
- **Dropbox** — **Dropbox** のデスクトップアプリをインストールして
  サインインします。
- **OneDrive** — Windows に組み込み。OneDrive を使っていればすでに同期
  しています。macOS では App Store から **OneDrive** をインストールして
  サインインします。

それでも設定にクラウドが現れない場合は、下の FAQ をご覧ください。

## 新しいコンピュータへの引っ越し

ライブラリはあなたについてきます。新しいコンピュータで：

1. クラウドのデスクトップアプリをインストールしてサインインし、同期が
   終わるのを待ちます。
2. Markwell をインストールします —
   [Markwell を入手する](../README.ja.md#markwell-を入手する)。
3. **設定**を開き、同じクラウドを選んで確認します。

Markwell はクラウドの中の同じ `Markwell` フォルダを指すようになります。
本も、ハイライトも、メモも、すべての保存コピーも、すでにそこにあります。
**ライブラリ**を開いて読むか、**履歴**で保存コピーを確かめてください。

## FAQ

**設定に自分のクラウドが出てきません。**
Markwell がクラウドを選択肢に出すのは、このコンピュータでそのクラウドの
同期フォルダを見つけられたときだけです。プロバイダのデスクトップアプリが
インストール済みでサインインしているか確認してください。クラウドフォルダを
特別な場所に置いている場合は、「**詳細設定：フォルダを自分で指定**」を
選び、その中のフォルダのフルパスを入力します — 例：
`/Users/you/Dropbox/Markwell`。

**実際に同期されるのは何ですか？**
Markwell フォルダの中身すべてです。`backups/`（保存コピー）、`output/`
（読めるページとエクスポート）、そして作成した `Markwell-archive-….zip`。
Markwell の小さな設定ファイルはライブラリの外（`~/.markwell/`）にあるので
同期されず、各コンピュータが自分の設定を持ちます — これは意図した設計です。

**Markwell が私のハイライトをどこかへアップロードすることはありますか？**
決してありません。Markwell はネットワーク接続を行いません。同期はすべて、
クラウドプロバイダ自身のアプリが、そのアプリのアカウントとルールのもとで
行います。クラウドアプリを止めれば、フォルダはただローカルに残るだけです。
